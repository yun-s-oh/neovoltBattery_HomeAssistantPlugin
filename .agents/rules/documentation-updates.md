# Documentation Updates Rule

**CRITICAL REQUIREMENT:** 
When making code or configuration changes to this repository, you MUST immediately update the following documentation files to reflect the new state of the project. Do not wait for the end of the conversation or for a prompt from the user.

1. **`.antigravity/context.md`**
   - Update the "Architecture", "Key Features", "Services Available", or "Recent Changes" sections if you introduce new files, services, endpoints, or architectural patterns.
   - Ensure the context always accurately represents the current capability of the integration.

2. **`.antigravity/logs/changelog.md`**
   - Add a detailed entry for the changes you just made under the current date.
   - Describe what was changed, added, or removed, and briefly explain why.

**Failure to update these files simultaneously with code changes is a violation of project rules.**
